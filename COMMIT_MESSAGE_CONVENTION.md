# 커밋 메시지 규격

이 저장소는 **Conventional Commits** 형식을 사용합니다.

## 기본 형식

```
<type>(<scope>): <summary>

<body>

<footer>
```

- `<type>`: 변경 유형(필수)
- `<scope>`: 변경 범위(선택, 모듈/폴더/기능명)
- `<summary>`: 50자 내외의 요약(필수, 마침표 생략 권장)
- `<body>`: 변경 이유/배경/주요 동작(선택)
- `<footer>`: 이슈/브레이킹 체인지 등 메타 정보(선택)

## Type 목록
- `feat`: 새로운 기능
- `fix`: 버그 수정
- `docs`: 문서 변경
- `style`: 포맷/스타일 변경(로직 변경 없음)
- `refactor`: 리팩터링(기능 변화 없음)
- `perf`: 성능 개선
- `test`: 테스트 추가/수정
- `build`: 빌드 시스템/의존성 변경
- `ci`: CI 설정 변경
- `chore`: 기타 변경
- `revert`: 되돌리기

## 권장 규칙
- 요약은 **명령형**으로 작성합니다. (예: "로그인 화면 오류 수정")
- 본문은 필요 시 추가하며, **왜** 변경했는지 설명합니다.
- 관련 이슈가 있다면 Footer에 `Refs: #123` 또는 `Closes: #123` 형식으로 표기합니다.
- 브레이킹 체인지가 있을 경우:
  - 제목 또는 Footer에 `!` 사용: `feat!: ...`
  - Footer에 `BREAKING CHANGE:` 설명 추가

## 예시

```
feat(auth): 소셜 로그인 추가

카카오 로그인 플로우를 추가하고 기존 로그인 UI를 정리했습니다.

Refs: #42
```

```
fix: 씬 전환 시 null 참조 오류 수정

BREAKING CHANGE: 관련 프리팹 참조 구조가 변경되었습니다.
```
